# 仕様: note.com 下書き投稿

> 関連ファイル: `scripts/post-to-note.ts`

---

## 機能概要 — v1 (2026-06-14)

`output/` の Markdown 記事を note.com に下書きとして投稿する。公開は手動で行う。

---

## 入力 / 前提条件

- `NOTE_API_TOKEN` 環境変数が設定されている（Bearer トークン、DevTools の Network タブで手動取得）
- 投稿対象の Markdown ファイルが存在する

## 処理フロー（正常系）

1. 投稿対象の Markdown ファイルを読み込む
2. frontmatter からタイトルを抽出する（`title:` → インライン形式 → Markdown 見出し の順で試行）
3. note.com API（非公開 API）に Bearer トークンで認証してリクエストを送信する
4. レスポンスから記事 ID / URL を取得してコンソールに出力する
5. 投稿済みフラグ（`noteUrl`）を `data/generated.json` に記録する

## 出力 / 事後状態

- note.com に下書き記事が1件作成される（公開状態ではない）
- コンソールに記事 URL が出力される

## 異常系・エッジケース

| 条件 | 期待される挙動 |
|------|--------------|
| `NOTE_API_TOKEN` が未設定 | 起動時にエラーを表示して終了 |
| タイトル抽出失敗 | 「タイトルを抽出できませんでした」を表示して中断 |
| note.com API エラー（4xx / 5xx） | ステータスコードとレスポンスボディを表示して終了 |
| トークン期限切れ（401） | 「トークンの再取得が必要です」と案内して終了 |

## スコープ外

- 自動公開（draft のみ）
- 投稿済み記事の更新・削除
- Playwright による UI 自動化（fetch API 直接呼び出しのみ）
