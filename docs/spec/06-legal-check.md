# 仕様: リーガルチェック

> 関連ファイル: `scripts/legal-check.ts`

---

## 機能概要 — v1 (2026-06-14)

生成された記事 Markdown を対象に、著作権侵害・名誉毀損・医療断言・過剰引用などの法的リスクを自動評価する。
パイプライン上の位置: `generate → legal-check → post-to-note`

---

## 入力 / 前提条件

- 生成済み Markdown ファイルのパス（`output/YYYY-MM-DD-*.md`）
- `GITHUB_TOKEN` 環境変数（AI 二次評価で使用）
- チェック実行は `pipeline.ts` から自動で呼び出される。単体実行も可（`--file` オプション）

---

## リスクレベル定義

| レベル | 意味 | パイプライン挙動 |
|-------|------|----------------|
| `low` | フラグなし | 警告なし・投稿続行 |
| `medium` | 軽微なフラグあり | コンソール警告・投稿続行 |
| `high` | 重大フラグあり | コンソールエラー・**投稿をブロック**（`post-to-note` をスキップ） |

---

## 処理フロー

```
1. ルールベース一次評価（正規表現・キーワード）
   → low  → 処理終了（フラグなし）
   → medium 以上 → ステップ 2 へ

2. AI 二次評価（LLM による精査）
   → legalRisk / legalFlags を確定

3. 結果を frontmatter に書き込み
4. High の場合は exit(1) で後続をブロック
```

---

## ルールベース一次評価（チェック項目）

### [R1] 医療・健康断言

**対象**: 全戦略

**検出パターン（正規表現）:**
```
/(〜に効く|〜を予防する|〜を治す|医学的に証明|科学的に証明|健康に良いことが証明|がん予防|血圧を下げる|ダイエット効果がある)/
```

**リスクレベル**: Medium（AI 精査で High に昇格可能）

---

### [R2] 企業批判・否定的記述（Industry Matrix 専用）

**対象**: Industry Matrix 戦略（frontmatter の `strategy: "industry-matrix"` で判定）

**検出パターン（企業名 + 否定語の近傍）:**
```
企業名から 100 文字以内に: (倒産|不祥事|詐欺|違法|訴訟|炎上|問題|危険|欠陥)
```

**リスクレベル**: Medium（AI 精査で High に昇格可能）

---

### [R3] 過剰引用

**対象**: 全戦略（News 戦略で特に注意）

**検出ロジック**: 鉤括弧 `「...」` または `"..."` で囲まれたテキストの総文字数が記事本文の 30% 超

**リスクレベル**: Medium

---

### [R4] 法的断定表現

**対象**: 全戦略

**検出パターン:**
```
/(という判決|という判断|が訴訟|は違法|は犯罪|に違反する)/
```

**リスクレベル**: High（AI 精査をスキップして即 High）

---

## AI 二次評価（Medium 以上のみ実行）

**モデル**: GitHub Models / GPT-4o（`baseURL: https://models.inference.ai.azure.com`）

**システムプロンプト骨子:**
```
あなたは日本の著作権法・不法行為法を踏まえたコンテンツリスク評価AIです。
以下の記事について、次の観点でリスクを評価してください。
- 著作権侵害の可能性（引用の適正性）
- 名誉毀損・信用毀損の可能性（特に実在企業・人物への言及）
- 医療・法律の断定的表現による誤情報リスク

出力: JSON { "riskLevel": "low|medium|high", "flags": string[], "summary": string }
```

**タイムアウト**: 30 秒（超過時は一次評価結果をそのまま採用）

---

## 出力 / 事後状態

### frontmatter への書き込み

```yaml
---
title: ...
legalRisk: "medium"
legalFlags: ["health-claim", "overquote"]
legalCheckedAt: "2026-06-14T10:00:00Z"
---
```

### フラグ識別子一覧

| フラグ | 対応するチェック |
|-------|----------------|
| `health-claim` | R1 医療・健康断言 |
| `company-negative` | R2 企業批判 |
| `overquote` | R3 過剰引用 |
| `legal-assertion` | R4 法的断定表現 |

### コンソール出力

```
[legal-check] ✓ low  — output/2026-06-14-foo.md
[legal-check] ⚠ medium [health-claim] — output/2026-06-14-bar.md
[legal-check] ✗ high  [company-negative, legal-assertion] — 投稿をブロックしました
```

---

## 異常系・エッジケース

| 条件 | 期待される挙動 |
|------|--------------|
| AI 二次評価が API エラー | 一次評価結果を採用してパイプライン続行（エラーログ出力） |
| AI 二次評価がタイムアウト | 同上 |
| frontmatter が壊れている | チェックをスキップ・警告のみ（ブロックしない） |
| `--skip-legal` フラグ | 全チェックをスキップ（手動実行時のエスケープ用） |

---

## スコープ外

- 外部の法律データベース・判例検索との連携
- 商標権の自動確認
- DMCA / 削除申請の自動処理
- チェック結果のダッシュボード / レポート出力
