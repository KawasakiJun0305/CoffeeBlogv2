# 仕様: 自動実行（GitHub Actions）

> 関連ファイル: `.github/workflows/pipeline.yml`

---

## 機能概要 — v1 (2026-06-14)

パイプライン（記事生成 → note.com 下書き投稿）を定期的に自動実行する。

---

## スケジュール

月・水・金 09:00 JST（cron: `0 0 * * 1,3,5`）

## 処理フロー（正常系）

1. GitHub Actions ワークフローが起動する
2. `GITHUB_TOKEN`（Actions 自動提供）と `NOTE_API_TOKEN`（Secrets）を環境変数に設定する
3. `auto` 戦略で記事を1件生成する
4. 生成成功時に note.com に下書き投稿する
5. 結果（成功/失敗、記事タイトル、URL）を Actions ログに出力する

## 異常系・エッジケース

| 条件 | 期待される挙動 |
|------|--------------|
| ワークフロー実行中にエラー | Actions の run が失敗ステータスになる（通知は GitHub デフォルト挙動） |
| `NOTE_API_TOKEN` が期限切れ | ワークフローが失敗し、ログにトークン再取得の案内を出力する |
