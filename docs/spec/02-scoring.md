# 仕様: 品質スコアリング

> 関連ファイル: `scripts/rank-articles.ts`

---

## 機能概要 — v1 (2026-06-14)

生成記事の品質を AI で評価し、低品質記事を識別できるようにする。

---

## 入力 / 前提条件

- `output/` に評価対象の Markdown ファイルが存在する
- `GITHUB_TOKEN` 環境変数が設定されている

## 処理フロー（正常系）

1. `output/` から最新 N 件（デフォルト: 5件）の Markdown ファイルを読み込む
2. 4軸で LLM に評価させる
3. 総合スコアと改善理由をコンソールに出力する

## 評価軸

| 軸 | 説明 | 配点 |
|----|------|------|
| `titleHook` | タイトルのクリック誘引性 | 25点 |
| `originality` | オリジナリティ・独自性 | 25点 |
| `concreteness` | 具体性（数値・地名・固有名詞） | 25点 |
| `readerFit` | 読者（20〜40代コーヒー初中級者）への適合度 | 25点 |

**合格基準**: 総合スコア 60点以上 / 100点

## 出力 / 事後状態

- 各記事のスコアと改善提案がコンソールに出力される
- スコアはファイルに保存しない（参考情報のみ）

## 異常系・エッジケース

| 条件 | 期待される挙動 |
|------|--------------|
| `output/` が空 | 「評価対象の記事がありません」を表示して終了 |
| スコアリング API エラー | エラーを表示し、そのファイルをスキップして続行 |
