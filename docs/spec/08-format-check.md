# 仕様: フォーマット・体裁チェック

> 関連ファイル: `scripts/format-check.ts`

---

## 機能概要 — v1 (2026-06-14)

生成された記事 Markdown の構造・体裁を検証し、読者体験を損なう可能性のある問題を検出する。
パイプライン上の位置: `generate → source-citation → fact-check → format-check → legal-check → post-to-note`

**挙動**: 常に続行（ブロックしない）。問題はコンソール警告のみ。

---

## 入力 / 前提条件

- 生成済み Markdown ファイルのパス（`output/YYYY-MM-DD-*.md`）
- 単体実行も可（`--file=<path>` オプション）

---

## チェック項目

### [P1] 本文の文字数

**基準:**

| 文字数 | 判定 |
|-------|------|
| 800字未満 | `too-short` 警告 |
| 800〜3,000字 | 正常 |
| 3,000字超 | `too-long` 警告（過剰に長い） |

**文字数カウント**: frontmatter と画像クレジット行を除く本文のみを対象

---

### [P2] 見出し構造

**チェック内容:**
- `## `（h2）が1個以上存在しない場合: `no-heading` 警告
- `# `（h1）が本文中に存在する場合: `h1-in-body` 警告（h1 は frontmatter の title を使用すること）

---

### [P3] タイトル

**チェック内容:**
- frontmatter の `title` が存在しない: `no-title` 警告
- `title` が 50 文字超: `title-too-long` 警告

---

### [P4] 過剰なリスト

**検出ロジック**: 連続する `-` または `*` リストアイテムが 10 行超
```
(\n[-*] .+){10,}
```

**判定**: `excessive-list` 警告（箇条書きに頼りすぎた構成の可能性）

---

### [P5] 空の見出し・段落

**検出ロジック**: `##` の直後が空行または次の `##` になっている場合

**判定**: `empty-section` 警告

---

## 出力 / 事後状態

### frontmatter への書き込み

```yaml
---
formatIssues: ["too-short", "no-heading"]
formatCheckedAt: "2026-06-14T10:00:00Z"
---
```

問題なし（全チェック通過）の場合:
```yaml
---
formatIssues: []
formatCheckedAt: "2026-06-14T10:00:00Z"
---
```

### フラグ識別子一覧

| フラグ | 対応するチェック |
|-------|----------------|
| `too-short` | P1 本文 800字未満 |
| `too-long` | P1 本文 3,000字超 |
| `no-heading` | P2 h2見出しなし |
| `h1-in-body` | P2 本文にh1あり |
| `no-title` | P3 titleフィールドなし |
| `title-too-long` | P3 title 50文字超 |
| `excessive-list` | P4 連続リスト10行超 |
| `empty-section` | P5 空の見出しセクション |

### コンソール出力

```
[format-check] ✓ 問題なし — output/2026-06-14-foo.md（1,234字）
[format-check] ⚠ 2件の体裁問題 [too-short, no-heading] — output/2026-06-14-bar.md（512字）
```

---

## 異常系・エッジケース

| 条件 | 期待される挙動 |
|------|--------------|
| frontmatter が壊れている | チェックをスキップ・警告のみ |
| `--skip-format` フラグ | 全チェックをスキップ |

---

## スコープ外

- AI による文章品質評価（流暢さ・可読性）
- 誤字脱字検出
- SEO スコアリング
